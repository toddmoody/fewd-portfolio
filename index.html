Detail Connect - Projects Map

This project was a great fit for a JavaScript integration with Google Maps API v3. There were a few key requirements and I ran into a few challenges along they way, here’s a brief summary...

  1. The client’s project data was stored in WorkFlow Max and although the were keen to fully automate they weren't quite ready. I wanted to build the application to read an unedited version of their data schema (in JSON-P) so a future automation would be painless.
  2. At any given time the client needs to show upwards of 90 projects across the country which will continue to grow. It was just too cramped using standard locator pins so I leveraged the JavaScript Google Marker Clustering library allowing for a much better UX, in particular on mobile.
  3. Not all jobs are the same! The client provides varying levels of services on each job and all of this needed to be dynamically displayed on the Google maps info-window (along with other attributes that are displayed for some service types and not others).
  4. Even though there is a substantial degree of individual job complexity there needed to be a very simple set of filters. This was refined and reiterated a lot down to a set of 4 filters and integrated with the marker clustering for intuitive regeneration of markers when filters are updated.
  5. Project images were crucial which they had stored in Dropbox (and displaying inside the native Dropbox window). A simple regex based solution was applied there to take the original URL query string and update it so the application had access to the raw image file which was then futher modified in the CSS.

Augusta Rise - single page web application

Existing client Barrett Homes needed a lightweight single page website to promote a new housing development - Augusta Rise. Initial low-fidelity wireframes confirmed that the initial build would be relatively static and simple, however potential future requirements would be more complex.

Additional (future) requirements include the integration of a JSON package including various construction related data (from multiple housing companies) with the idea being that multiple additional UI elements would be rolled out providing a much richer user experience. Additionally, it’s likely individual property listings will be added likely involving integration with multiple external API’s.

The current production build leverages a bootstrap, vanilla JS, & jQuery which allowed me to meet a time-sensitive launch deadline. As a ‘phase two’, I’m currently rebuilding this single page application in React using Create React App and React-Bootstrap. Please check out progress at my GitHub Repo.

MyMango - Flexbox pricing table

I’ve been applying Flexbox wherever possible for a while now. To me, it feels like a huge improvement for positioning elements and even creating grid based layouts using CSS. So, after I recently worked on a Pricing Table project for MyMango recently that required using floats (due to their legacy browsers) I decided to do a post-rebuild using Flexbox.

I think it’s a big improvement not only on the front-end but also from code maintainability perspective. Albeit, Flexbox browser compatibility isn’t quite 100% it’s getting better and hopefully we’ll soon be waving goodbye to traditional floats, clear fixes etc or at least be using less of them.

I’d like to enhance this project by adding in a JS filter in order for users to toggle between monthly and annual pricing and also drop in a few more UI elements. Check out my GitHub repo for more details.

Stay Code Fit - React App

I keep learning new things about code every day but the only constant (excuse the pun) is that code typically doesn’t go hand-in-hand with keeping fit and healthy. Inherently, as developers we spending a lot of time in front of the computer - lets call it an occupational hazard.

The objective of this app is about trying to make some incremental changes so that time spent (in front and away from the desk) is more healthy and potentially even more productive. I have in mind relevant Meetup groups, twitter feeds, and videos ranging from ergonomic workstation set-up through to rapid workouts.

I’m building this project in React and bootstrapping it using Create React App. I plan to incorporate various UI elements (many of which will integrate with external API’s) so it feels like a good fit to leverage efficient and separate React components. I’m currently working on low-fidelity mockups followed by an initial MVP prototype. Check out my GitHub Repo for progress.
